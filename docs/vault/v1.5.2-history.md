### 2.4 persist v3 -> v4 migrate 链沉淀, theme 字段透传, 旧 v1/v2 数据可升级

v1.5.2 升级 useSettingsStore persist v3 -> v4, 新增 totalSecondsToday + lastSessionDate 字段. migrate 函数:

```typescript
if (version < 4) {
  return {
    ...persistedState,
    totalSecondsToday: persistedState.totalSecondsToday ?? 0,
```

theme / llm / difficulty 字段全保留, 不重置. 旧 v1/v2/v3 数据加载后:
- theme 字段缺失 -> 默认 'light' (与 v1.5.1 视觉一致, 0 跳变)
- totalSecondsToday 缺失 -> 默认 0 (今日累计为 0)
- lastSessionDate 缺失 -> 默认 null (下次阅读触发 resetTodayIfNewDay)

migrate 链沉淀, 后续 v1.5.3/v1.6.0 加字段时, 仅加 version 升级 + 透传旧字段.

## 3. 失败教训 (2 点)

### 3.1 v1.5.1 Stage 1 dispatch 用了拼写错误的 subagent_type "general_potion_task", 浪费 1 轮

v1.5.1 Stage 1 派发 subagent 时, 误把 subagent_type 写成 "general_potion_task" (应是 "general-purpose" 或 "general_purpose"), 系统报 "Tool result missing due to internal error", 浪费 1 轮 dispatch. v1.5.2 Stage 1-4 派发时, 严格使用 "general-purpose" 类型, 0 派发错误.

**经验**: 派发 subagent 前, 一定要校验 subagent_type 拼写. 常用 3 种: "general-purpose" (默认) / "statusline-setup" / "Explore" (调研). 其它类型不存在.

### 3.2 主页 Hero 注入"今日已读"后 copy margin-bottom 微调, 视觉节奏与原版一致但需留意

Contract 28 在 HeroSection 注入"Heute bereits X Min. gelesen" 后, 副标题 copy 与新注入行的 margin-bottom 需要微调 (从 0.5rem 改为 0.25rem, 避免视觉密度过大). 视觉节奏与 v1.5.1 Hero 区域整体保持一致, 但需留意:
- 注入行 a11y aria-label 用德文全称 "Heute bereits X Minuten gelesen" (不用缩写)
- 注入行视觉字号与副标题一致 (clamp 0.875-1rem)
- 注入行在桌面上左侧 60% 区域, 移动端 stack 时单独一行

**经验**: 主页 Hero 区域视觉节奏敏感, 注入新元素后必须 mobile + tablet + desktop 三视口实测, 避免 60/40 split 失衡.
